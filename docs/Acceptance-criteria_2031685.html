Created by Henrique Pureza, last modified by Luca Gonzalez on Nov 10, 2022

Terminology

  • Acceptance criteria: a set of requirements that must be met in order to mark a user story complete. Determines the scope of each user story

  • User Story: see User stories

Format

  • Given [condition]… When [event]… Then [result]

  • Each criteria refers to one user story. A user story can have multiple acceptance criteria

History

  • Acceptance Criteria will be added before each coding sprint such that user stories of the sprint can be tested in reference to the criteria.
