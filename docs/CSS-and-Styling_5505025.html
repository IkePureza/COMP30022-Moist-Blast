docs : CSS and Styling

Created by Keigo Nagai on Aug 17, 2022

UI/UX is an important element for our project and especially in the field of web development.

The team had a discussion to decide whether to use (or rely more heavily on) Tailwind CSS or standard CSS. Below are the pros and cons of both, supported by external resources to back each argument. Material UI is listed below as a potential alternative.

Tailwind CSS

Tailwind is a CSS framework that have prebuilt classes that can be used flexibly to build components, directly in markup.

https://tailwindcss.com/

Pros

Cons

  • Control over styling

  • Faster CSS styling process

  • Ease of responsive design

  • Popular in the industry

  • Styling and HTML are mixed (inline styling)

  • Takes additional time to learn

  • Goes against coding practices in previous subjects to separate styling and HTML layout.

Material UI

MUI is a library of prebuilt (and styled) components to build a UI/page. It may facilitate customizable styling, but its main purpose is providing UI components.

https://mui.com/

Pros

Cons

  • More than styling, design principles and guidelines from Google are applied, making pages comprehensive

  • Motion and animation is easy and enforced in designs

  • Prebuilt components are easy to deploy - styling is already done

  • Popular in industry (more so than TailwindCSS)

  • Since originally built for Android and Google mobile phones, designs will feel like a native app

  • Lack of creativity and customization in styling components

  • Motion and animation may be costly and unnecessary in some situations
